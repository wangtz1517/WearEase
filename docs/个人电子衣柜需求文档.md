# 个人电子衣柜需求文档

## 1. 项目概述

### 1.1 项目名称
个人电子衣柜 / Personal Digital Wardrobe

### 1.2 项目背景
用户希望建立一个可长期维护的个人衣物管理系统，用于记录所有衣服、查看衣物状态、辅助整理收纳，并基于现有衣物自动生成穿搭建议。系统需要支持拍照上传或上传商品链接，尽量自动完成衣物识别、信息归档和标准化展示，减少手工录入成本。

### 1.3 项目定位
这不是一个单纯的“相册”或“购物清单”，而是一个围绕衣物生命周期设计的个人管理平台，兼顾以下三个目标：

1. 衣物资产管理：知道自己“有什么、在哪、适合什么时候穿”。
2. 穿搭辅助决策：基于已有衣橱生成可执行的日常搭配建议。
3. 季节性收纳辅助：按季节筛出当季衣物，帮助进行换季整理。

## 2. 产品目标

### 2.1 核心目标
1. 降低衣物信息整理成本，让新增一件衣服的归档过程尽量简单。
2. 提供清晰、统一的衣物展示方式，避免照片杂乱难以比对。
3. 帮助用户快速找到适合当前季节、场景和天气的衣服。
4. 基于已有衣物自动输出穿搭方案，提高衣橱利用率。

### 2.2 产品成功标准
1. 用户可以在 1 分钟内完成一件新衣物的基础录入。
2. 大部分衣物可以自动生成基础分类信息，用户只需少量修正。
3. 用户能够按季节、类别、颜色、正式程度等维度快速筛选衣物。
4. 系统生成的穿搭建议具备可读性和可执行性，不只是简单随机组合。

## 3. 目标用户

### 3.1 核心用户
有衣物整理需求、愿意长期维护个人衣橱数据的个人用户。

### 3.2 用户特征
1. 衣服数量较多，记不清具体有哪些单品。
2. 有换季收纳习惯，希望只保留当季衣物在外。
3. 日常会纠结穿什么，希望系统给出搭配建议。
4. 愿意通过上传照片、商品链接等方式维护衣橱。

## 4. 核心使用场景

### 4.1 新衣归档
用户买了新衣服后，通过拍照或粘贴商品链接导入系统，系统自动识别衣物信息，并生成标准展示图，用户只需补充或修正细节。

### 4.2 日常找衣服
用户按类别、颜色、季节、场景等条件快速搜索自己的衣物，确认可穿选择。

### 4.3 自动穿搭
用户选择季节、场景或目标风格后，系统根据衣橱内现有单品输出穿搭方案。

### 4.4 换季整理
系统自动识别适合当前季节的衣物，输出“当季应保留在手边的衣服”和“可收纳入柜的衣服”清单。

## 5. 功能需求

## 5.1 衣物录入与归档

### 5.1.1 新增方式
系统应支持以下新增方式：

1. 本地上传照片
2. 手机拍照后上传
3. 粘贴商品链接导入
4. 手动新建衣物条目

### 5.1.2 自动识别能力
系统可基于上传图片或商品页面信息，自动提取以下内容：

1. 衣物大类：上衣、裤子、外套、裙子、鞋、包、配饰等
2. 细分类：衬衫、T 恤、卫衣、牛仔裤、西装裤、羽绒服等
3. 主颜色和辅色
4. 图案特征：纯色、条纹、格纹、印花等
5. 材质或厚度判断：轻薄、常规、厚实
6. 适穿季节：春、夏、秋、冬，可多选
7. 适用场景：通勤、休闲、运动、正式、居家、旅行等
8. 风格标签：基础、简约、街头、商务、复古等

### 5.1.3 标准正视图生成
系统应支持将原始照片或商品图处理为“标准衣物展示图”，用于统一展示。

标准图要求：

1. 尽量以正视角度展示衣物主体
2. 弱化或移除复杂背景
3. 尺寸、留白、构图尽量统一
4. 用于衣柜列表页和详情页展示

说明：
第一版可先实现“自动裁切主体 + 背景简化 + 统一比例”的轻量方案，不要求达到电商级抠图精度。后续版本再增强为更高质量的标准正视图。

### 5.1.4 手动编辑与校正
用户应可手动修改自动识别结果，包括：

1. 衣物名称
2. 分类和标签
3. 季节属性
4. 颜色
5. 收纳位置
6. 是否保留 / 是否闲置 / 是否已淘汰
7. 备注信息

### 5.1.5 归档状态
每件衣物应有明确状态：

1. 正常在衣橱中
2. 当季常用
3. 已收纳
4. 待清洗
5. 待处理 / 闲置
6. 已淘汰 / 已送出

## 5.2 衣物信息管理

### 5.2.1 衣物基础信息字段
建议每件衣物至少包含以下字段：

1. 衣物 ID
2. 名称
3. 主图
4. 原始图片
5. 一级分类
6. 二级分类
7. 颜色
8. 图案
9. 材质 / 厚度
10. 季节标签
11. 场景标签
12. 风格标签
13. 收纳位置
14. 购买时间
15. 购买渠道 / 来源链接
16. 价格（可选）
17. 穿着频率（后续统计）
18. 最近穿着时间（后续统计）
19. 当前状态
20. 备注

### 5.2.2 查询与筛选
系统应支持以下筛选条件组合：

1. 按分类筛选
2. 按季节筛选
3. 按颜色筛选
4. 按场景筛选
5. 按风格筛选
6. 按收纳状态筛选
7. 按是否常穿 / 闲置筛选

### 5.2.3 展示方式
建议提供两种展示方式：

1. 卡片视图：更适合浏览和搭配
2. 列表视图：更适合管理和批量编辑

## 5.3 自动穿搭功能

### 5.3.1 搭配目标
系统根据用户现有衣物自动生成穿搭建议，避免重复购买和搭配困难。

### 5.3.2 输入条件
用户可选择以下条件后生成搭配：

1. 当前季节
2. 场景：上班、出门、约会、旅行、居家等
3. 风格偏好：基础、极简、休闲、正式等
4. 天气冷暖程度（后续可结合天气）
5. 是否优先使用最近较少穿的衣物

### 5.3.3 输出内容
每套搭配至少包含：

1. 上装
2. 下装或连体类单品
3. 外套（如需要）
4. 鞋 / 包 / 配饰（可选）
5. 搭配说明

### 5.3.4 搭配逻辑要求
系统应尽量遵循以下原则：

1. 季节匹配正确
2. 场景匹配正确
3. 颜色不过度冲突
4. 单品风格相对统一
5. 避免推荐已收纳或不在当前可穿清单中的衣物

说明：
第一版建议先做“规则型搭配推荐”，即基于分类、季节、场景和颜色规则进行组合；后续版本再加入更强的 AI 风格理解和个性化偏好学习。

## 5.4 季节衣物管理

### 5.4.1 当季识别
系统应能根据当前季节自动给出“推荐保留在外的衣物”列表。

### 5.4.2 换季清单
系统应生成两份清单：

1. 当季建议拿出来的衣物
2. 当前可收纳起来的非当季衣物

### 5.4.3 辅助整理能力
系统可从以下角度辅助用户整理：

1. 标记长期未穿单品
2. 标记不适合当前季节的单品
3. 统计各季节衣物数量是否失衡
4. 提醒某类单品是否过多或过少

## 5.5 收纳与位置管理

### 5.5.1 收纳位置
系统应支持记录衣物存放位置，例如：

1. 衣柜左区
2. 抽屉 1
3. 换季箱 A
4. 鞋柜下层

### 5.5.2 使用价值
收纳位置功能要服务于两个目标：

1. 快速找到实物
2. 换季时知道哪些衣服已被收起

## 5.6 后续可扩展功能
以下功能不建议放入第一版，但应在架构上预留空间：

1. 与天气联动，按当地天气推荐穿搭
2. 穿搭日历，记录每天穿了什么
3. 衣物使用频率统计
4. 购物建议，识别衣橱缺口
5. 胶囊衣橱模式
6. 出行打包清单
7. 多用户 / 家庭共享衣柜

## 6. 页面与信息架构建议

为了后续网页设计，建议第一版先包含以下页面：

1. 首页 / 仪表盘
展示衣物总数、当季衣物数、待整理数、推荐搭配、换季提醒。

2. 我的衣柜
展示所有衣物，支持筛选、搜索、卡片查看和批量管理。

3. 新增衣物
支持上传照片、输入链接、手动录入，并显示自动识别结果。

4. 衣物详情页
展示标准图、原始图、分类标签、收纳位置、季节信息、编辑入口。

5. 穿搭推荐页
按季节、场景、风格生成搭配，并支持收藏或重新生成。

6. 季节整理页
展示“本季保留”和“建议收纳”的衣物清单。

7. 设置页
管理分类规则、风格标签、收纳位置、季节策略等。

## 7. 用户流程建议

### 7.1 新衣录入流程
1. 用户上传照片或商品链接
2. 系统自动识别衣物信息
3. 系统生成标准展示图
4. 用户确认或修正分类与标签
5. 用户填写收纳位置
6. 保存进入衣柜

### 7.2 日常穿搭流程
1. 用户进入穿搭推荐页
2. 选择季节、场景和风格
3. 系统生成若干套可选搭配
4. 用户收藏或继续刷新

### 7.3 换季整理流程
1. 系统识别当前季节
2. 自动列出建议保留的衣物
3. 自动列出建议收纳的衣物
4. 用户确认并调整衣物状态

## 8. MVP 建议范围

第一版建议聚焦“能真正用起来”的最小闭环，不要一开始做得过重。

### 8.1 MVP 必做
1. 衣物上传与新建
2. 图片基础归档
3. 自动识别基础分类与季节标签
4. 手动编辑分类和标签
5. 我的衣柜列表页
6. 衣物详情页
7. 基于规则的穿搭推荐
8. 当季衣物筛选与换季清单

### 8.2 MVP 可弱化处理
1. 标准正视图先做简化版
2. 商品链接解析先支持主流电商链接或仅做占位
3. 自动搭配先不追求高级审美评分
4. 收纳管理先做单层位置字段，不做复杂空间结构

### 8.3 暂缓到后续版本
1. 天气联动
2. 穿搭历史统计
3. 高级 AI 风格学习
4. 多端同步与账号系统

## 9. 非功能需求

### 9.1 易用性
1. 支持移动端上传图片
2. 页面操作尽量简洁，减少重复录入
3. 支持批量修改标签和状态

### 9.2 性能
1. 衣柜列表在中等规模数据下保持流畅
2. 上传图片后应尽快返回预览和识别结果

### 9.3 数据与隐私
1. 衣物照片和个人穿搭偏好属于隐私数据
2. 若后续接入云端或 AI 服务，需要明确图片处理和数据存储方式

## 10. 风险与难点

1. 图片来源复杂，自动识别准确率可能不稳定。
2. “标准正视图生成”对图像处理能力要求较高，建议分阶段实现。
3. 自动穿搭的用户满意度高度依赖分类质量和规则设计。
4. 如果一开始字段太多，可能导致用户录入负担过重。

## 11. 产品设计建议

### 11.1 设计方向
网页整体应更像“个人衣物管理工作台”，而不是普通电商页面。核心体验应突出：

1. 视觉整洁，方便统一查看衣物
2. 信息结构清晰，便于快速筛选
3. 录入流程顺滑，减少操作负担
4. 推荐结果实用，而非炫技

### 11.2 设计原则
1. 优先展示标准化衣物卡片
2. 优先保证“找得到”和“看得懂”
3. AI 功能要以“可修正”为前提，不应黑盒化

## 12. 下一步建议

如果本需求文档方向正确，下一步建议进入网页设计阶段，按以下顺序推进：

1. 确定产品信息架构
2. 确定首页与核心页面布局
3. 确定视觉风格与组件体系
4. 输出首页、衣柜页、新增页、搭配页的初版原型
