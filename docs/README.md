# Documentation Index

这个目录保存项目的产品、AI、设计和历史资料。

现在文档目录已经按主题拆分成独立子目录，适合直接放在 GitHub 上浏览。

## 目录结构

```text
docs/
├─ README.md
├─ product/
├─ ai/
├─ design/
└─ archive/
```

## 各子目录说明

### `product/`

保存产品需求、场景定义、功能边界等文档。

### `ai/`

保存新增衣物标准化、AI 调用链路、Provider 方案等文档。

### `design/`

保存命名、页面方向、视觉和交互说明。

### `archive/`

预留给历史版本、废弃方案、阶段性记录。

## 说明

这次整理的目标是：

1. 保留你原有详细文档
2. 让 GitHub 首页和 `docs/` 入口更清楚
3. 减少后续继续扩文档时的混乱

如果后面继续扩展，建议新文档优先按子目录放置，而不是继续平铺在 `docs/` 根目录。
