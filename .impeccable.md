## Design Context

### Users
这个产品面向多人使用的个人衣橱管理场景，而不是只服务单一拥有者。

核心用户通常具备这些特征：
- 衣物数量较多，需要长期维护自己的衣橱数据
- 希望快速知道“我有什么、现在能穿什么、接下来该怎么搭”
- 愿意通过上传照片、商品链接或手动补充信息来维护衣橱
- 需要在“管理衣物”和“日常穿搭决策”之间来回切换

典型使用场景：
- 日常快速找衣服、做穿搭决定
- 新购入衣物的拍照上传、AI 标准化与归档
- 浏览、筛选和整理现有衣橱
- 按季节做收纳和轮换
- 回看最近穿搭并复用有效搭配

这个产品必须是可长期真实使用的系统，而不是一次性演示页。界面要让用户感到省力、清晰、可靠。

### Brand Personality
品牌气质关键词：
- 温暖
- 高级
- 丝滑

延伸特质：
- 克制而不冷淡
- 有秩序但不僵硬
- 精致但不炫耀
- 像“个人衣橱工作台”而不是“购物网站”

整体体验应该像一个私人衣物工作室与归档空间，不能呈现出交易感、促销感、后台感或炫技感。

### Aesthetic Direction
当前产品应保持“暖色、编辑感、atelier 风格的衣橱工作台”方向。

应该接近的感觉：
- 温暖中性色和轻微染色的界面表面
- 节制但有品质感的字体系统
- 明确的层级和有呼吸感的留白
- 让衣物图片与穿搭结果成为视觉主体
- 平滑、安静、解释性的动效

明确避免：
- 任何一眼看上去很像 AI 自动生成模板的设计
- 电商促销页气质
- 科技后台气质
- 潮牌官网气质
- 过于少女化的风格

未来需要支持深色模式，因此颜色、对比、阴影、边框、层级都必须建立在可迁移的设计 token 之上，而不是临时值。

### Design Principles
1. 衣物是主角。界面是为了支持用户理解衣物、搭配衣物、管理衣物，而不是抢视觉中心。
2. 优先降低理解成本。用户进入任一页面后，应尽快知道“这里是什么、我现在能做什么、下一步是什么”。
3. 保持系统语言一致。相同层级的字体、按钮、间距、动画、反馈方式必须在全站统一。
4. 动效要解释状态变化。动画应该帮助理解切换、展开、加载和反馈，并且始终保持丝滑而不花哨。
5. 设计要支持长期演进。任何页面级优化，都要能沉淀为未来全站可复用的规则。

### Layout And Adaptation Rules
以下规则属于硬性约束，而不是风格建议：

1. 每个页面都必须占满可用屏幕
- 页面进入后应形成完整的全屏工作界面
- 页面骨架需要优先使用 viewport 高度和稳定的内层容器组织结构
- 不能让页面看起来像“内容只占一小块，剩下一大片空白”

2. 页面必须对不同屏幕和窗口变化稳定适配
- 浏览器放大、缩小、改变比例时，布局不能错位、重叠、溢出、乱码或出现模块漂移
- 不同尺寸屏幕上都要保持清晰层级和完整可用性
- 响应式不是简单缩放，而是根据空间变化重新组织信息结构

3. 页面必须根据内容类型选择最佳呈现方式
- 短操作和关键控制：优先固定、贴边、工具条化、就近放置
- 长列表和历史记录：优先垂直滚动区域、分段列表、分页或懒加载
- 图片密集内容：优先网格、横向滑轨或对比结构
- 双栏工作台：优先将“操作区”和“结果区”解耦，必要时只让其中一栏滚动
- 长表单或多步骤流程：优先分段、折叠、步骤化，而不是一次性堆满

4. 滚动策略必须由内容驱动
- 不要默认整页滚动
- 应根据模块职责决定是整页滚动、局部滚动、横向滑动、固定区、弹窗还是抽屉
- 每一个滚动行为都必须服务于“更稳定、更省力、更美观的浏览和操作”

5. 排版稳定性优先级高于装饰
- 任何视觉优化都不能以牺牲排版稳定、内容可读、点击精度为代价
- 不允许出现文本换行失控、控件重叠、图片裁切错误、滚动区域冲突等问题

### Module Presentation Rules
在做任何页面或模块优化时，必须先判断模块内容，再决定展示方式：

- 导航类模块：强调方向性、稳定性、低认知负担
- 数据概览类模块：强调层级、快速扫读、强主次关系
- 衣物卡片类模块：强调图片主体、名称可读、对齐节奏
- 筛选和控制类模块：强调密度控制、状态清晰、操作连续性
- 生成结果类模块：强调结果主视觉、控制就近、历史可回溯
- 表单和输入类模块：强调步骤感、错误恢复、低填写压力

如果一个模块同时承担多种职责，应先拆职责，再决定布局，而不是把不同内容强行塞进同一个视觉壳里。

### Working Rules For Future UI Changes
- 优先使用系统化 design tokens，而不是零散的临时值
- 相同角色的标题、正文、标签、按钮必须统一
- 相同角色的动画时长、缓动、状态切换必须统一
- 先判断信息结构，再决定视觉表达
- 先判断内容长度和密度，再决定滚动方式
- 先判断屏幕约束，再决定布局拆分方式
- 优先做“稳定、耐看、长期可维护”的设计，而不是一次性的视觉炫技
- 避免一切会立即读成“AI 味很重”的设计动作
